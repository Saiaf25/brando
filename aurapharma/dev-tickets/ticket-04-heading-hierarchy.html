Issue

The H1 tag is the single strongest on-page signal a page can give about its topic. It is also the heading that screen readers announce when a user lands on a page. Five of Aura's six core templates ship with zero H1 elements in the rendered HTML.

Other heading-hierarchy problems on the homepage

Beyond the missing H1, the homepage has three additional issues:

  1. Duplicate H2s. "Foot Care Cream", "Vitamin C Serum", and "Hyaluronic Acid serum" each appear twice in the H2 list. The same product is featured in two different sections, and both sections use the product name as an H2. Re-use a more general H2 for the second section ("Featured products", "Customer favourites", etc.).
  2. Decorative H3s. The "Our process in four easy steps" section uses H3 elements that contain the literal text "1", "2", "3", "4". Headings should describe content, not be visual labels. The step numbers should be styled spans inside an H3 that names the step, or moved out of the heading hierarchy entirely.
  3. Inconsistent capitalisation in headings. Some product H1s on detail pages are in ALL CAPS ("CLEANSER", "ANTI AGING SERUM", "BRIGHTENING CREAM"), others are Title Case ("Lip Balm", "Vitamin C"). Pick a single style and enforce it at the data layer (the product name field) so the H1 inherits the canonical casing.

Required heading structure per template

/ (Home)

H1: "Aura Pharma | Cosmeceutical and wellness products in Qatar"
  H2: "Our finest selection"
  H2: "Discover our quality products"
  H2: "Discover our sets"
  H2: "Our process in four easy steps"
    H3: "Add to cart"
    H3: "Sign in"
    H3: "Pay"
    H3: "Get delivered"
  H2: "Contact Aura Pharma"

The bare digits 1 to 4 should be styled spans inside the step H3, not standalone H3 text.

/about

H1: "About Aura Pharma" (promoted from H2)
  H2: "Vision"
  H2: "Mission"
  H2: "Value"

/contactus

H1: "Contact Aura Pharma" (or "Get in touch")

This page is short. One H1 is enough.

/faq

H1: "Frequently asked questions"
  H2: each question topic

/shop

H1: "Aura Pharma products" (or category-driven once categories exist)
  H2: "Featured" / "All products"
  Sidebar labels "Sort by" and "Categories" should be styled as labels, not H2.

Product pages (currently correct)

H1: Product name in canonical casing (Title Case, e.g. "Vitamin C Serum")

Adjust the product name field at the data layer for the four ALL-CAPS products: CLEANSER, SUN Screen, ANTI AGING SERUM, BRIGHTENING CREAM.

Fix

  1. Edit each template (Home, About, Contact, FAQ, Shop) so that the page topic is wrapped in an H1 element, exactly once per page.
  2. Demote the heading currently doing the job of an H1 (where it exists). On About this is "About Aura Pharma" (currently H2); change its tag to H1 in the template.
  3. Replace the decorative number-only H3s on the homepage with H3s that name each step, and move the digit into a styled span.
  4. De-duplicate the H2s on the homepage product sections by giving each section a section-level heading instead of repeating the product name.
  5. Standardise product name casing in the data layer so the H1 on every product page renders in Title Case.
Do not solve this with CSS only (e.g., styling an H2 to look like an H1). Search engines and assistive technologies read the tag, not the visual style.

Verification

  • For each affected URL, run in DevTools console: document.querySelectorAll('h1').length. Expected: 1.
  • Run document.querySelectorAll('h1')[0].textContent and confirm the text describes the page topic.
  • Open each page with VoiceOver (Mac) or NVDA (Windows) and confirm the first heading announced names the page.
  • Re-run a Screaming Frog crawl with the H1 filter set. Expected: zero pages flagged "H1: Missing".
  • Check the homepage for duplicate H2s in DevTools: [...document.querySelectorAll('h2')].map(h => h.textContent.trim()). Expected: no duplicates.

Why this is critical

For an 18-URL site competing for category-level keywords ("vitamin C serum Qatar", "cosmeceutical brand Doha"), the H1 is the page's first chance to tell Google what it is about. Sending the homepage with no H1 is sending a page with no topic statement. Add to that the inconsistent product casing and the decorative H3 numbers, and the heading hierarchy on this site is doing the opposite of what it is supposed to do, which is help both search engines and assistive technologies parse the page in seconds. The fix is template-level and ships once.
