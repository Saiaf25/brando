Issue

The site currently outputs zero JSON-LD structured data on every page. Confirmed on home, about, contact, faq, shop, and the Vitamin C product page. For a pharma and cosmeceutical ecommerce brand, structured data is the primary mechanism Google uses to understand what the business is, what it sells, where it operates, and how the products relate to each other.

Without it, Aura is invisible to:

  • Google Knowledge Panel for the brand entity.
  • Product rich results in SERPs (price, availability, rating).
  • FAQ rich results from the existing /faq page.
  • Sitelinks search box on branded queries.
  • Local business listings for the Qatar office.

Reference JSON-LD blocks

Treat each block as a template. Replace {{...}} placeholders with real data at render time. Output one combined <script type="application/ld+json"> per page (a JSON array) or multiple separate scripts. Both are valid per Google's docs. Never hard-code values that exist in the data layer.

1. Organization (sitewide)

Place in the base template so it renders on every page.

{
  "@context": "https://schema.org",
  "@type": "Organization",
  "@id": "https://www.aurapharma.org/#organization",
  "name": "Aura Pharma",
  "url": "https://www.aurapharma.org/",
  "logo": "https://www.aurapharma.org/web/image/website/1/logo",
  "description": "Aura Pharma is a leading company in non-pharmacological and cosmeceutical products in Qatar.",
  "address": {
    "@type": "PostalAddress",
    "addressCountry": "QA",
    "addressLocality": "{{city}}",
    "streetAddress": "{{street}}",
    "postalCode": "{{postal_code}}"
  },
  "contactPoint": [{
    "@type": "ContactPoint",
    "telephone": "{{phone_e164}}",
    "contactType": "customer service",
    "email": "{{contact_email}}",
    "areaServed": "QA",
    "availableLanguage": ["en", "ar"]
  }],
  "sameAs": [
    "{{facebook_url}}",
    "{{instagram_url}}",
    "{{linkedin_url}}",
    "{{twitter_url}}"
  ]
}

Note: logo currently resolves to aurapharma.odoo.com in the live OG tags. Make sure the value above uses the production domain (aurapharma.org), not the legacy subdomain. The sameAs array should include only profiles that exist (see ticket AURA-003).

2. WebSite + SearchAction (homepage only)

{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "@id": "https://www.aurapharma.org/#website",
  "url": "https://www.aurapharma.org/",
  "name": "Aura Pharma",
  "publisher": { "@id": "https://www.aurapharma.org/#organization" },
  "inLanguage": "en",
  "potentialAction": {
    "@type": "SearchAction",
    "target": {
      "@type": "EntryPoint",
      "urlTemplate": "https://www.aurapharma.org/shop?search={search_term_string}"
    },
    "query-input": "required name=search_term_string"
  }
}

Only ship the SearchAction if the search URL pattern is real and returns results for any query. Do not ship a fake search endpoint.

3. Product + Offer (every product page)

{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "{{product_name}}",
  "description": "{{product_description}}",
  "image": [
    "{{primary_image_url}}",
    "{{secondary_image_url}}"
  ],
  "sku": "{{sku}}",
  "brand": {
    "@type": "Brand",
    "name": "Aura Pharma"
  },
  "offers": {
    "@type": "Offer",
    "url": "{{canonical_product_url}}",
    "priceCurrency": "QAR",
    "price": "{{price}}",
    "availability": "https://schema.org/InStock",
    "itemCondition": "https://schema.org/NewCondition",
    "seller": { "@id": "https://www.aurapharma.org/#organization" }
  }
}

All 9 products are currently priced at 1.00 QAR (placeholder). Do not ship Product schema until real prices are set, or Google will index the placeholder price. See master sheet row 92.

Generate the array dynamically from the product collection so it stays in sync as products are added or removed.

5. FAQPage (/faq)

{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "{{question_1_text}}",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "{{answer_1_text}}"
      }
    },
    {
      "@type": "Question",
      "name": "{{question_2_text}}",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "{{answer_2_text}}"
      }
    }
  ]
}

Only include questions that are actually visible to the user on /faq. Hidden Q&A is a Google policy violation.

6. BreadcrumbList (every non-home page, once breadcrumbs ship)

{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Home", "item": "https://www.aurapharma.org/" },
    { "@type": "ListItem", "position": 2, "name": "Shop", "item": "https://www.aurapharma.org/shop" },
    { "@type": "ListItem", "position": 3, "name": "{{product_name}}", "item": "{{canonical_product_url}}" }
  ]
}

Depends on visible breadcrumb UI being added first (see master sheet row 46). The structured data must mirror what is on the page.

Implementation order

  1. Ship Organization first in the base template. It is the lowest-risk addition and unlocks brand entity recognition for every page on the site.
  2. Hold Product schema until the placeholder 1.00 QAR pricing is replaced with real values. Shipping it now would index the wrong prices.
  3. Ship FAQPage as soon as the FAQ content is finalised. It is one of the fastest paths to a SERP rich result.
  4. Ship LocalBusiness on Contact once the Qatar office address, phone number, and hours are confirmed by the client.
  5. Ship BreadcrumbList together with the breadcrumb UI work in master sheet row 46.
  6. Ship ItemList on Shop any time after Organization is live.

Verification

  • Validate every template with the Google Rich Results Test. Expected: zero errors, warnings acknowledged.
  • Validate with the Schema.org Validator for full schema-spec compliance.
  • In DevTools console: document.querySelectorAll('script[type="application/ld+json"]').length. Expected: at least 1 on every page, more on product / FAQ / shop pages.
  • Once Search Console is connected, confirm the Enhancements section starts surfacing Organization, Product, FAQ, and Breadcrumb reports without errors.
  • Search site:aurapharma.org in Google a few weeks after deploy. Expected: rich results begin appearing for FAQ and Product entries.

Why this is critical

Aura is a small, new ecommerce brand competing in a category with dominant retailers and aggregators. Structured data is one of the few levers a small brand has that does not require backlinks or domain authority to work. Every other ticket fixes something that is broken; this one builds something that does not yet exist. Without it, the site relies on Google guessing what each page is about from text alone, which for a 100-word product page is a losing strategy.
