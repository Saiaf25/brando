AURA-003 · Sitewide 404s on the footer social-link block

Issue

The footer of every page on the site renders four social-media icons that link to internal stub URLs the CMS provides as a redirect-to-social-profile shortcut. None of the four stubs are configured, so all four return HTTP 404. The footer block is included in every template (verified on home, product, about, contact, shop). For an 18-URL site, this means 4 broken internal links on every page, totalling more than 200 broken link relations across the crawl.

HTTP/2 404
content-type: text/html;charset=utf-8
server: Odoo.sh

Root cause

The CMS exposes /website/social/<platform> as a built-in redirect helper. When a real social-profile URL is configured for a platform, the helper 301-redirects to that profile. When no URL is configured (the current state), the helper returns 404 instead of being hidden. The footer template renders icons regardless of configuration state, so users and crawlers hit the failure path.

Fix

Decide whether Aura Pharma has active social profiles to link to. Two clean paths:

Path A: Wire the icons to real profiles

If the brand has live profiles on Facebook, Twitter/X, LinkedIn, Instagram (or any subset):

  1. Configure the public profile URL for each platform in the CMS social-settings panel.
  2. Verify the helper now 301-redirects from /website/social/<platform> to the real profile URL with HTTP status 301 (not 302) and an https:// destination.
  3. Render only the icons for platforms that have a configured URL. Hide icons for platforms without one.

Path B: Remove the social-link block

If the brand does not maintain social profiles right now:

  1. Remove the social-link block from the footer template entirely. Do not leave dead icons.
  2. Remove any duplicate rendering of the block on the contact page.
  3. Re-introduce the block once social profiles exist.
Per the audit context, Aura Pharma is a recently launched brand on its own domain. If social profiles are planned but not yet created, ship Path B now and switch to Path A once the profiles are live. Leaving 4 visible-but-broken social icons on every page signals an abandoned brand.

Verification

  • Run curl -sI https://www.aurapharma.org/website/social/facebook. Expected: either HTTP 301 with Location: https://www.facebook.com/<handle> (Path A) or HTTP 404 with the link no longer present in the rendered HTML (Path B).
  • Re-crawl the site with Screaming Frog (or any crawler). Expected: zero internal 4xx hits in the response codes report.
  • Inspect any production page footer in the browser. Expected: no broken icons, no href="/website/social/..." attributes pointing at unconfigured platforms.
  • Once the fix ships, ask Search Console (after it is connected) to revalidate. Expected: no Not found (404) entries in Pages > Not indexed for these URLs.

Why this is critical

This is not just an SEO finding. Every visitor who clicks a social icon to find Aura on Instagram or LinkedIn lands on a 404 page. That is a direct user-experience failure on the most-clicked footer element. Combined with the placeholder copy on About and Contact, and the 1.00 QAR placeholder pricing on every product, the abandoned-social signal makes the site read as unfinished. Fixing it costs minutes and removes the worst trust signal on the page.
